01개요

회사가 만들려는 것을,
가상 데이터로 먼저 만들어 봤습니다.

무엇을 만들었는지 네 가지로 정리했습니다.

대화형 Jarvis

Slack 같은 협업툴에서 "현대건자재 미수금?" 하고 물으면, ERP를 조회해 답합니다. 앞의 대화 맥락들을 기억하고, 모르는 건 지어내지 않습니다.

반복 업무 자동화 3종

반복 업무 중에서 자동화가 가능한 시나리오를 골라 구현했습니다. 안전재고 발주 추천, 미확인 입금 매칭, 견적서 텍스트에서 품목·단가를 뽑아 구매전표 초안 만들기. 최종 실행 권한은 사람에게 있습니다.

일에 맞는 처리 방식 선택

순서가 정해진 일은 코드로 자동 처리하고(workflow), 무슨 질문이 올지 모르는 영역은 AI가 스스로 도구를 골라 쓰게 하고(agent), 여러 부서를 한 번에 보는 분석은 분석가 여럿을 동시에 돌렸습니다(multi-agent).

실제 API 연동

원래는 이카운트 ERP와 Flow로 구현하려 했지만, 데모 단계라 만들 수 있는 범위에 한계가 있었습니다. 그래서 무료로 쓸 수 있는 Airtable을 ERP 대신, Slack을 Flow 협업툴 대신 연결해, 같은 구조를 실제 API로 구현했습니다.

전 과정 가상의 데이터로 구현했습니다.

02대화형 Jarvis · 실제 화면

아는 것만 답하고,
모르는 건 지어내지 않습니다.

AI의 가장 큰 약점은 모르는 것도 그럴듯하게 지어내는 환각(hallucination)입니다. Jarvis는 하네스 엔지니어링으로 이걸 막았습니다. 데이터를 조회하는 읽기 전용 도구만 주고, 답은 반드시 실제 조회 결과에 근거하게 하고, 도구로 확인되지 않으면 지어내지 말고 모른다고 답하도록 시스템 프롬프트에 규칙을 넣었습니다.

발주나 입금 매칭처럼 데이터를 직접 바꾸어야 하는 부분은 사람이 직접 승인해야 하기에, 데이터를 바꾸는 도구 자체를 넣진 않았습니다. 실제로 처리할 때는 Ksjarvis가 보내준 내용을 기반으로 사람이 승인하고 결정합니다.

03멀티에이전트 아침 브리핑

매일 아침, 팀별 채널로 알아서 정리됩니다.

재고·재무·미수를 각각 보는 분석가 셋이 병렬로 돌아 현황을 만들고, 팀별 채널로 자동 발송됩니다. 매일 오전 9시에 자동으로 올라올 수 있도록 서버에 예약 작업(cron)을 걸어 뒀습니다.

병렬 분석
재고·재무·미수 분석가 셋을 동시에 실행(asyncio · 파이썬에서 여러 작업을 한꺼번에 돌리는 방식)한 뒤 하나로 종합
자동 발송
각 영역을 담당 채널로, 종합본은 #jarvis로. 매일 오전 9시 예약 발송
관측·평가
Jarvis가 어떤 도구를 어떤 순서로 썼는지 전부 기록(trace)하고, 정답을 미리 정해둔 테스트 질문 5개로 자동 채점해 5개 모두 통과
실연동
데이터 Airtable 실 API · 협업툴 Slack(Socket Mode + Web API) · 서버 systemd 24시간
04만들면서 신경 쓴 것

잘 돌아가는 것보다, 틀리지 않는 것을 먼저 생각했습니다.

안전하게
모르거나 확실하지 않은 것은 답변하지 않고, 발주·입금 같은 실제 실행은 사람이 합니다. 에이전트에는 조회 도구만 주고 데이터를 바꾸는 도구는 넣지 않았습니다
비용과 정확도
순서가 정해진 워크플로는 코드로 돌리고, 판단이 필요한 곳에만 에이전트를 씁니다. SKU 매칭처럼 코드가 더 정확하고 싼 곳엔 LLM을 쓰지 않고, 추론·분류·검증은 작업에 맞춰 더 싼 모델로 분기하도록 설계했습니다 (Sonnet·Haiku·GPT-4o-mini)
검증할 수 있게
어떤 도구를 어떤 순서로 썼는지 다 기록해 둬서, 틀린 답이 나오면 어디서 잘못됐는지 되짚을 수 있습니다. 또 테스트 질문으로 Jarvis의 답을 검증했습니다
05직접 만든 자동화

반복되는 일은, 전부 자동화해 왔습니다.

가상자산 투자와 마케팅 프리랜서를 하면서, 시세 모니터링·차익(아비트라지) 탐색·온체인 데이터 수집·알림, 리서치 제작 등 매번 손이 가던 일들을 자동화했습니다. 2026년 1월부터 시작하여 지금까지 총 19개의 프로젝트를 진행했습니다.

예측시장 분석 봇
Polymarket의 가격·뉴스·이상 거래를 실시간으로 모니터링·분석 (3종) · asyncio + WebSocket · 누적 약 34,446건 분석, 데이터 약 7.6GB 수집
예측시장 델타뉴트럴 포인트파밍 봇
3개월 집중 개발 · 다중 API 연동, 설정값 158개로 전략 튜닝
게임 자동화
화면을 캡처해 OpenCV 템플릿 매칭·색 기반(HSV)으로 인식하고 플레이를 자동화 (여러 게임) · 한 봇 기준 18일 / 267 commits · 기성 라이브러리 조합
온체인·에어드랍 자동화
에어드랍 참여, 온체인 데이터 수집, 차익(아비트리지) 기회 모니터링 등
RPOW 암호화폐 채굴 자동화
PC CPU로 채굴 자동화
알림 허브 + 인프라
여러 봇의 동작 상태·이슈를 자동으로 받아보는 텔레그램 알림 봇(bongbis) 직접 개발 · AWS 서버 + 자동 재시작(systemd) + 예약 작업(cron) + GitHub 15개 저장소 자동 동기화

이 외에도 미션 자동화, 온체인 이상거래 감지, 차익거래 모니터링, 브라우저 자동화 등 총 19개의 프로젝트를 진행했습니다.
